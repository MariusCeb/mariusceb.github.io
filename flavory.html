What

Flavory is a full-stack web application built to make food and wine pairing accessible to everyone. The landing page is structured around three distinct sections: an editorial hero with the Flavory wordmark and tagline ("Where Every Flavor Finds Its Perfect Pairing"), a 150vh scroll-animation section where a cinematic wine sequence plays frame by frame tied to scroll position, and a discovery section with editorial copy and a primary CTA leading into the application itself.

The application runs on a Node.js/Express backend that handles user authentication with JWT tokens and bcryptjs password hashing, cookie-based session management, and a Stripe payment integration — indicating a premium pairing engine behind the marketing surface. The frontend is built in pure HTML, CSS, and vanilla JavaScript, using Playfair Display typography and a warm crimson-cream palette (#b22222 / #fff2f2) that reflects the subject matter.

Users navigate the site via a sticky top navbar with logo, three section links (Home, Menu, Wine), and a Login CTA button. A custom crimson dot cursor and red text-selection highlight reinforce the brand identity down to the micro-interaction level.

Why

Wine pairing is widely perceived as arcane expertise — the domain of sommeliers, not everyday diners. The vocabulary alone (tannins, minerality, terroir) creates a barrier that makes even an enthusiastic home cook hesitate to pick a bottle with confidence. Flavory's premise is that this knowledge can be encoded into software and served on demand.

The landing page needed to carry the premium weight of the subject. A standard product page felt inadequate — wine is visual, sensory, and emotional. The decision to centre the page on a 300-frame scroll-scrubbed animation was deliberate: it communicates craft and seriousness before the user reads a single word. The Stripe integration signals that the pairing engine is a real service worth paying for, not a toy demo.

Typographically, Playfair Display was chosen over a sans-serif specifically for its editorial, high-contrast letterforms — the same typeface you'd find on a wine label or a fine-dining menu. Every visual decision reinforces the same message: this product respects its subject.

Frame-by-Frame Scroll Animation

300 PNG frames are preloaded into an array of Image objects at page initialization — eagerly, not lazily, so the animation is never caught waiting mid-scroll. A requestAnimationFrame loop runs continuously, reading the current scrollY value and dividing it by the total scrollable height to produce a normalized progress value (0–1). This maps directly to a frame index (0–299), and the computed Image object's src is assigned to the visible <img> element. The video section is set to 150vh — taller than the viewport — giving a generous scroll range that translates into a smooth, unhurried playback speed. The result is a cinematic wine-pour sequence that the user literally controls with their hand.

Dual-Panel Intersection Observer Reveal

The two content panels inside the scroll-animation section — "L'arte del vino?" on the left and the Flavory description on the right — begin their lives off-screen: the left panel is positioned at translateX(-100%) with opacity 0, the right at translateX(100%) with opacity 0. An IntersectionObserver watches the video section container. When the container crosses the viewport threshold, both panels receive the .show class simultaneously, triggering a 1-second CSS ease transition that brings them to translateX(0%) and opacity 1. When the section leaves the viewport on scroll-back, the class is removed and the panels retreat — the animation is fully reversible and replays every time the section enters view.

Node.js / Express Backend with Auth & Stripe

Behind the landing surface sits a complete Express server. The authentication layer uses bcryptjs for password hashing at registration and comparison at login, with jsonwebtoken generating signed JWT tokens on success. cookie-parser reads and writes session cookies across requests. A dotenv configuration layer keeps environment secrets (JWT secret, Stripe keys, database credentials) out of the codebase. The Stripe integration adds a payment layer — suggesting that the core pairing engine, personalized recommendations, or saved pairing history are gated behind a subscription or one-time purchase. nodemon handles hot-reloading during development.

Outcome

What shipped

A complete marketing landing page and backend scaffolding: sticky navbar with logo, three navigation links and a Login CTA; a full-viewport hero section with the Flavory wordmark, tagline and branded typography; a 150vh scroll-animation section with 300-frame cinematic playback and dual slide-in content panels; a discovery section with editorial copy, image-and-text layout, and a primary CTA button; a copyright footer. The Node.js/Express backend includes user registration and login routes, JWT token generation and verification middleware, bcryptjs password hashing, cookie-parser session management, Stripe payment integration endpoints, and a dotenv environment configuration layer. A custom crimson dot cursor and red selection highlight complete the brand identity across the full page experience.

What I learned

Preloading 300 images at page load is deceptively simple in code but consequential in production. Each new Image() call initiates a fetch immediately — on a slow mobile connection, this floods the network before the user sees anything meaningful. A production-grade version would need progressive frame buffering (preload only the next N frames in a sliding window) or a dedicated frame-streaming CDN strategy. I also identified a subtle inefficiency: the requestAnimationFrame loop runs on every tick regardless of whether scrollY has changed, causing unnecessary re-renders even when the page is idle. The fix is a frame-skip guard: compare the new computed frame index against the last rendered one and bail early if they match. These are the optimisations that separate a polished product from a working prototype.

On the design side, this project reinforced the power of typographic consistency as a branding signal. Playfair Display, the crimson accent, and the cream background aren't decorative choices — they're trust-building. Every element speaks the same visual language, and that coherence is what makes the product feel premium before you even read the copy. The micro-details — cursor colour, text-selection highlight, hover state transitions — compound into an experience that feels intentional from edge to edge.
